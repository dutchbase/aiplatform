Create `app/sitemap.ts` following Next.js 14 App Router pattern for dynamic sitemaps.

**Implementation:**
- Export default function returning MetadataRoute.Sitemap (array of sitemap entries)
- Include existing routes: `/`, `/login`, `/profiel`
- Include planned routes from Phase 06: `/openclaw`, `/openclaw/installatie`, `/openclaw/tutorials`, `/blog`, `/qa`, `/ai-assistenten`, `/kennisbank`
- For dynamic routes ([slug], [id]): Include only overview pages (/blog, /qa) since no data exists yet
- Each entry includes:
  - `url`: Full absolute URL using metadataBase pattern
  - `lastModified`: Current date (new Date())
  - `changeFrequency`: 'weekly' for content pages, 'monthly' for static pages
  - `priority`: 1.0 for home, 0.8 for main sections, 0.6 for sub-pages
- Use TypeScript with proper types: `import type { MetadataRoute } from 'next'`

**Example structure:**
```typescript
import type { MetadataRoute } from 'next'

export default function sitemap(): MetadataRoute.Sitemap {
  const baseUrl = process.env.NEXT_PUBLIC_BASE_URL || 'http://localhost:3000'

  return [
    {
      url: baseUrl,
      lastModified: new Date(),
      changeFrequency: 'weekly',
      priority: 1.0,
    },
    // ... other routes
  ]
}
```

**Why this approach:**
- Next.js 14 automatically serves this at /sitemap.xml
- Dynamic function allows future expansion to query database for blog/qa slugs
- changeFrequency and priority help search engines understand content importance
- TypeScript ensures type safety with MetadataRoute.Sitemap

<task type="auto">
  <name>Task 2: Implement robots.txt</name>
  <files>app/robots.ts</files>
  <action>
Create `app/robots.ts` following Next.js 14 App Router pattern for dynamic robots.txt.

**Implementation:**
- Export default function returning MetadataRoute.Robots object
- Allow all user agents to crawl all content: `userAgent: '*'`, `allow: '/'`
- Disallow /api routes: `disallow: ['/api/']` (API endpoints shouldn't be indexed)
- Include sitemap reference with absolute URL
- Use TypeScript with proper types: `import type { MetadataRoute } from 'next'`

**Example structure:**
```typescript
import type { MetadataRoute } from 'next'

export default function robots(): MetadataRoute.Robots {
  const baseUrl = process.env.NEXT_PUBLIC_BASE_URL || 'http://localhost:3000'

  return {
    rules: {
      userAgent: '*',
      allow: '/',
      disallow: ['/api/'],
    },
    sitemap: `${baseUrl}/sitemap.xml`,
  }
}
```

**Why this approach:**
- Next.js 14 automatically serves this at /robots.txt
- Explicit disallow for /api prevents indexing of API routes
- Sitemap reference helps crawlers discover all URLs
- TypeScript ensures type safety with MetadataRoute.Robots
  </action>
  <verify>
Run `npm run dev`, visit http://localhost:3000/robots.txt, confirm valid robots.txt with User-agent: *, Allow: /, Disallow: /api/, and Sitemap: reference.
  </verify>
  <done>
/robots.txt returns valid robots.txt allowing all content except /api/, with sitemap reference pointing to absolute URL.
  </done>
