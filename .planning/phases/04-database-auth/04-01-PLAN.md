  <action>
Create the directory `supabase/migrations/` and the file `00001_profiles.sql`.

The SQL file must contain these sections IN ORDER:

**Section 1: Profiles table**
```sql
create table public.profiles (
  id uuid not null references auth.users(id) on delete cascade,
  display_name text,
  role text not null default 'user' check (role in ('user', 'moderator', 'admin')),
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),
  primary key (id)
);
```
- Use CHECK constraint for role values (NOT PostgreSQL ENUM). CHECK constraints are easier to modify later without a full migration.
- `on delete cascade` ensures profile cleanup when a user is deleted from auth.users.
- `role` defaults to `'user'` -- new signups are always regular users.

**Section 2: Enable Row Level Security**
```sql
alter table public.profiles enable row level security;
```

**Section 3: Auto-create profile on signup trigger**
```sql
create or replace function public.handle_new_user()
returns trigger
language plpgsql
security definer set search_path = ''
as $$
begin
  insert into public.profiles (id, display_name)
  values (new.id, new.raw_user_meta_data ->> 'display_name');
  return new;
end;
$$;

create trigger on_auth_user_created
  after insert on auth.users
  for each row execute procedure public.handle_new_user();
```
CRITICAL: `security definer set search_path = ''` is REQUIRED.
- `security definer` lets the function bypass RLS (it runs as the postgres role, not the calling user). Without this, the INSERT into profiles would be blocked by RLS.
- `set search_path = ''` prevents search_path injection attacks. Use fully qualified table names (public.profiles) inside the function body.
- `raw_user_meta_data ->> 'display_name'` reads the display_name from signup metadata passed via `options.data`.

**Section 4: Auto-update updated_at trigger**
```sql
create or replace function public.update_updated_at_column()
returns trigger
language plpgsql
as $$
begin
  new.updated_at = now();
  return new;
end;
$$;

create trigger update_profiles_updated_at
  before update on public.profiles
  for each row execute procedure public.update_updated_at_column();
```

**Section 5: RLS Policies**

Policy 1 - Authenticated users can read all profiles:
```sql
create policy "Profiles are viewable by authenticated users"
  on public.profiles
  for select
  to authenticated
  using ( true );
```
WHY `true`: All authenticated users need to see display names (for Q&A posts, comments). Anonymous users are excluded by targeting `authenticated` role only.

Policy 2 - Users can update their own profile:
```sql
create policy "Users can update their own profile"
  on public.profiles
  for update
  to authenticated
  using ( (select auth.uid()) = id )
  with check ( (select auth.uid()) = id );
```
CRITICAL: Use `(select auth.uid())` NOT `auth.uid()` directly. The SELECT wrapper causes PostgreSQL to cache the function result per-statement instead of per-row, improving performance from 179ms to 9ms on large tables.

Policy 3 - Admins can update any profile:
```sql
create policy "Admins can update any profile"
  on public.profiles
  for update
  to authenticated
  using (
    exists (
      select 1 from public.profiles
      where id = (select auth.uid())
      and role = 'admin'
    )
  );
```

NO INSERT policy for authenticated users: Only the trigger inserts profiles.
NO DELETE policy: Profiles are deleted via CASCADE when the auth user is deleted.
