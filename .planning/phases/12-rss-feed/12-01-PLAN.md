  <action>
Create the directory `app/feed.xml/` and the file `app/feed.xml/route.ts`.

This is a Next.js App Router Route Handler. It must:

1. Import `blogPosts` from `lib/data/blog`. The BlogPost type (created in Phase 11) will have at minimum: `slug: string`, `title: string`, `description: string`, `publishedAt: string` (ISO date string). Use these fields. If Phase 11 added an `excerpt` field, prefer it over `description` for the item description.

2. Read the base URL from `process.env.NEXT_PUBLIC_BASE_URL || 'https://aiassistentenhub.nl'`. Do NOT hardcode localhost as the fallback; use the production domain as the fallback so that committed XML references a real domain.

3. Build the RSS 2.0 XML string. Required structure:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<rss version="2.0" xmlns:atom="http://www.w3.org/2005/Atom">
  <channel>
    <title>AI Assistenten Hub</title>
    <link>{baseUrl}</link>
    <description>Nieuws, tutorials en updates over AI-assistenten in het Nederlands.</description>
    <language>nl</language>
    <atom:link href="{baseUrl}/feed.xml" rel="self" type="application/rss+xml"/>
    {items}
  </channel>
</rss>
```

4. Sort blogPosts by `publishedAt` descending (newest first). Take the first 20 items only.

5. For each post, build an `<item>`:

```xml
<item>
  <title>{escapeXml(post.title)}</title>
  <link>{baseUrl}/blog/{post.slug}</link>
  <description>{escapeXml(post.description)}</description>
  <pubDate>{new Date(post.publishedAt).toUTCString()}</pubDate>
  <guid isPermaLink="true">{baseUrl}/blog/{post.slug}</guid>
</item>
```

6. Implement an `escapeXml` helper function (private, not exported) that replaces:
   - `&` with `&amp;` (must be first to avoid double-escaping)
   - `<` with `&lt;`
   - `>` with `&gt;`
   - `"` with `&quot;`
   - `'` with `&apos;`

7. Return a `Response` (not NextResponse) with the XML string and headers:
   ```typescript
   { 'Content-Type': 'application/rss+xml; charset=utf-8' }
   ```

8. Export the function as `GET`:
   ```typescript
   export async function GET(): Promise<Response> { ... }
   ```

Do NOT use `NextResponse` — use the native `Response` constructor which works correctly for non-JSON Route Handlers. Do NOT import from `next/server` unless already present in the project.

TypeScript: all variables must be typed. The return type is `Promise<Response>`. No `any`.

**app/layout.tsx — add RSS alternate link to existing metadata:**

The existing `metadata` export in `app/layout.tsx` already has an `alternates` object with `canonical` and `languages`. Add `types` to it:

```typescript
alternates: {
  canonical: '/',
  languages: {
    'nl-NL': '/',
  },
  types: {
    'application/rss+xml': `${process.env.NEXT_PUBLIC_BASE_URL || 'https://aiassistentenhub.nl'}/feed.xml`,
  },
},
```

This causes Next.js to render `<link rel="alternate" type="application/rss+xml" href="...">` in the HTML `<head>` automatically. Do NOT manually add a `<link>` tag in the JSX — the metadata API handles it.

| Veld | Bron |
|------|------|

| link | {baseUrl}/blog/{BlogPost.slug} |
| description | BlogPost.description (XML-escaped) |

```typescript

  },
}
```

Next.js rendert dit als `<link rel="alternate" type="application/rss+xml">` in elke pagina's `<head>`.

- Route Handler (GET), geen Server Component
- Data-bron: `lib/data/blog.ts` (statische array)
- Maximaal 20 items, gesorteerd op `publishedAt` aflopend
- XML special characters worden escaped: `&`, `<`, `>`, `"`, `'`
- `atom:link` self-referentie voor feed-validators
