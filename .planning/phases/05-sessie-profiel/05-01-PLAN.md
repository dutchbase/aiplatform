**File 1: Create `app/profiel/actions.ts`** -- Server Action for profile updates.

```typescript
'use server'

import { revalidatePath } from 'next/cache'
import { createClient } from '@/lib/supabase/server'
```

Define and export a type `ProfileFormState`:
```typescript
export type ProfileFormState = {
  message: string | null
  error: string | null
}
```

Export an async function `updateProfile` with signature:
```typescript
export async function updateProfile(
  prevState: ProfileFormState,
  formData: FormData
): Promise<ProfileFormState>
```

CRITICAL: The first parameter MUST be `prevState` (not formData) because `useFormState` passes the previous state as the first argument. This is the #1 mistake with useFormState.

Implementation:
1. Create server client: `const supabase = await createClient()`
2. Get user: `const { data: { user }, error: authError } = await supabase.auth.getUser()`
3. If `authError` or `!user`, return `{ message: null, error: 'Je bent niet ingelogd.' }`
4. Extract `displayName` from formData: `const displayName = formData.get('display_name') as string`
5. Validate: if `!displayName || displayName.trim().length === 0`, return `{ message: null, error: 'Weergavenaam mag niet leeg zijn.' }`
6. Update profile: `const { error } = await supabase.from('profiles').update({ display_name: displayName.trim() }).eq('id', user.id)`
7. If `error`, return `{ message: null, error: 'Er is een fout opgetreden bij het opslaan.' }`
8. Call `revalidatePath('/profiel')` to refresh the page with updated data
9. Return `{ message: 'Profiel succesvol bijgewerkt.', error: null }`

Do NOT:
- Wrap redirect() calls here (there are none -- this action returns state, not redirects)
- Use Zod validation (simple string check suffices for display_name in this phase)
- Call `supabase.auth.updateUser()` to sync raw_user_meta_data (profiles table is single source of truth)
- Chain `.select()` after `.update()` (we only need to check for error, not return data)

**File 2: Create `app/profiel/profile-form.tsx`** -- Client Component for the edit form.

```typescript
'use client'

import { useFormState, useFormStatus } from 'react-dom'
import { updateProfile, type ProfileFormState } from './actions'
```

CRITICAL: Import `useFormState` from `react-dom` (NOT `useActionState` from `react`). This project uses React 18.3.1. `useActionState` is React 19+ only.

Create a separate `SubmitButton` component inside this file:
```typescript
function SubmitButton() {
  const { pending } = useFormStatus()
  return (
    <button
      type="submit"
      disabled={pending}
      className="rounded-md bg-primary px-4 py-2 text-sm font-medium text-primary-foreground hover:bg-primary/90 disabled:opacity-50"
    >
      {pending ? 'Opslaan...' : 'Opslaan'}
    </button>
  )
}
```

`SubmitButton` MUST be a separate component (not inline) because `useFormStatus` can only read form status from a CHILD component of the form, not from the form component itself.

Export `ProfileForm` component that accepts props `{ displayName: string; userId: string }`:

1. Define initial state: `const initialState: ProfileFormState = { message: null, error: null }`
2. Use `useFormState`: `const [state, formAction] = useFormState(updateProfile, initialState)`
3. Render a `<form action={formAction}>` with:
   - Label "Weergavenaam" (`htmlFor="display_name"`) styled with `block text-sm font-medium mb-1`
   - `<input id="display_name" name="display_name" type="text" defaultValue={displayName} required />` styled with `w-full rounded-md border border-input bg-background px-3 py-2 text-sm`
   - Use `defaultValue` (NOT `value`) -- this creates an uncontrolled input that allows editing without an `onChange` handler
   - Error message: if `state.error`, show `<p>` with `text-sm text-destructive bg-destructive/10 p-3 rounded-md`
   - Success message: if `state.message`, show `<p>` with `text-sm text-green-600 bg-green-50 p-3 rounded-md dark:bg-green-950 dark:text-green-400`
   - `<SubmitButton />`
4. Wrap form content in `className="space-y-4"`

The `userId` prop is accepted for potential future use but is not needed in the current form (the Server Action determines the user from the session).

**File 3: Create `app/profiel/page.tsx`** -- Protected profile page (Server Component).

```typescript
import { redirect } from 'next/navigation'
import { createClient } from '@/lib/supabase/server'
import type { Metadata } from 'next'
import { ProfileForm } from './profile-form'
```

Add metadata export:
```typescript
export const metadata: Metadata = {
  title: 'Mijn profiel',
  description: 'Bekijk en bewerk je profiel op de AI Assistenten Hub.',
}
```

Export default async function `ProfielPage()`:

1. Create server client: `const supabase = await createClient()`
2. Get user: `const { data: { user }, error } = await supabase.auth.getUser()`
3. If `error` or `!user`, call `redirect('/login')` -- CRITICAL: Do NOT wrap this in try/catch. Next.js `redirect()` works by throwing. A try/catch will swallow it and the user will see an empty profile page instead of being redirected.
4. Fetch profile: `const { data: profile } = await supabase.from('profiles').select('display_name, role').eq('id', user.id).single()`
5. Render the page:
   - Container: `<div className="container max-w-2xl py-10">`
   - Heading: `<h1 className="text-3xl font-bold mb-6">Mijn profiel</h1>`
   - Email section: `<div className="mb-6">` with `<p className="text-sm text-muted-foreground">E-mailadres</p>` and `<p>{user.email}</p>`
   - Profile form: `<ProfileForm displayName={profile?.display_name ?? ''} userId={user.id} />`

Do NOT:
- Add `'use client'` directive (this is a Server Component)
- Use `await` on `searchParams` (Next.js 14 pattern -- synchronous, but this page doesn't use searchParams anyway)
- Show the user's role (out of scope per research -- role management is for later phases)
- Use `getSession()` instead of `getUser()`
