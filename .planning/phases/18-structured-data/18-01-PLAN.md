  <action>
Add a `<script type="application/ld+json">` tag to `QAVraagPage`. The schema uses `@type: "QAPage"` with the question as `mainEntity` and the answers as `suggestedAnswer` items.

**Types available in scope:**
- `question`: `Question` — has `title` (string), `body` (string), `created_at` (string), `profiles: { display_name: string | null } | null`
- `answersWithReplies`: `Array<Answer & { replies: AnswerReply[] }>` — each answer has `body` (string), `created_at` (string), `profiles`

Build the schema object after `answersWithReplies` is resolved (it is already computed via `Promise.all` before the return), then wrap the return in a Fragment:

```typescript
const qaSchema = {
  '@context': 'https://schema.org',
  '@type': 'QAPage',
  mainEntity: {
    '@type': 'Question',
    name: question.title,
    text: question.body,
    answerCount: answersWithReplies.length,
    dateCreated: question.created_at,
    suggestedAnswer: answersWithReplies.map((answer) => ({
      '@type': 'Answer',
      text: answer.body,
      dateCreated: answer.created_at,
      url: `${process.env.NEXT_PUBLIC_BASE_URL ?? 'http://localhost:3000'}/qa/vraag/${question.id}`,
    })),
  },
}

return (
  <>
    <script
      type="application/ld+json"
      dangerouslySetInnerHTML={{ __html: JSON.stringify(qaSchema) }}
    />
    <div className="container mx-auto px-4 py-8">
      {/* ...existing JSX unchanged... */}
    </div>
  </>
)
```

Place the `qaSchema` variable definition immediately before the `return` statement, after the `answersWithReplies` declaration and `Promise.all` call.

When there are zero answers, `suggestedAnswer` will be an empty array `[]` — this is valid Schema.org (answerCount: 0, no suggestedAnswer items). Do not conditionally omit the script tag.

**Important:** `process.env.NEXT_PUBLIC_BASE_URL` is already used in `app/layout.tsx` and `app/sitemap.ts` — this is the established pattern for the production URL.
