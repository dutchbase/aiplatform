Create `lib/qa/queries.ts` with four async read functions. Each function creates its own Supabase client via `createClient()` from `@/lib/supabase/server`. Do NOT import from supabase-js directly. Each function returns typed data (using types from `@/lib/qa/types`). On Supabase error, throw with the error message so callers handle it at the Server Component level.

Function signatures and behaviour:

1. `getQuestions(limit = 20): Promise<Question[]>`
   - Query: `supabase.from('questions').select('*').order('created_at', { ascending: false }).limit(limit)`
   - If error throw new Error(error.message)
   - Return data as Question[]

2. `getQuestionById(id: string): Promise<Question | null>`
   - Query: `supabase.from('questions').select('*').eq('id', id).single()`
   - If error.code === 'PGRST116' (PostgREST "no rows") return null
   - If other error throw new Error(error.message)
   - Return data as Question

3. `getAnswersByQuestionId(questionId: string): Promise<Answer[]>`
   - Query: `supabase.from('answers').select('*').eq('question_id', questionId).order('created_at', { ascending: true })`
   - If error throw new Error(error.message)
   - Return data as Answer[]

4. `getRepliesByAnswerId(answerId: string): Promise<AnswerReply[]>`
   - Query: `supabase.from('answer_replies').select('*').eq('answer_id', answerId).order('created_at', { ascending: true })`
   - If error throw new Error(error.message)
   - Return data as AnswerReply[]

Table names come from Phase 13 schema: `questions`, `answers`, `answer_replies`.

Define three Zod schemas (if zod available):
```typescript
const createQuestionSchema = z.object({
  title: z.string().min(10, 'Titel moet minimaal 10 tekens zijn'),
  body: z.string().min(20, 'Beschrijving moet minimaal 20 tekens zijn'),
})

const createAnswerSchema = z.object({
  question_id: z.string().uuid('Ongeldig vraag-ID'),
  body: z.string().min(10, 'Antwoord moet minimaal 10 tekens zijn'),
})

const createReplySchema = z.object({
  answer_id: z.string().uuid('Ongeldig antwoord-ID'),
  body: z.string().min(5, 'Reactie moet minimaal 5 tekens zijn'),
})
```

Implement `createQuestion(formData: FormData): Promise<{ error?: string; id?: string }>`:
1. `const supabase = await createClient()`
2. `const { data: { user } } = await supabase.auth.getUser()`
3. If `!user` return `{ error: 'Niet ingelogd' }`
4. Parse title and body from formData (as string)
5. Validate with createQuestionSchema.safeParse — on failure return `{ error: result.error.errors[0].message }`
6. Insert: `supabase.from('questions').insert({ title, body, user_id: user.id }).select('id').single()`
7. On DB error return `{ error: 'Er is iets misgegaan. Probeer het opnieuw.' }` (never expose raw DB error to caller)
8. `revalidatePath('/qa')`
9. Return `{ id: data.id }`

Implement `createAnswer(formData: FormData): Promise<{ error?: string; id?: string }>`:
1-3. Same auth check as above.
4. Parse question_id and body from formData
5. Validate with createAnswerSchema.safeParse
6. Insert into `answers` table: `{ question_id, body, user_id: user.id }`
7. On DB error return generic error message
8. `revalidatePath('/qa')` and `revalidatePath(`/qa/vraag/${question_id}`)`
9. Return `{ id: data.id }`

Implement `createReply(formData: FormData): Promise<{ error?: string; id?: string }>`:
1-3. Same auth check.
4. Parse answer_id and body from formData
5. Validate with createReplySchema.safeParse
6. Insert into `answer_replies` table: `{ answer_id, body, user_id: user.id }`
7. On DB error return generic error message
8. `revalidatePath('/qa')` — no specific path since we don't know question_id here
9. Return `{ id: data.id }`

Important: never use `redirect()` inside these actions — they return data, not navigate. Never expose stack traces or Supabase error codes to the return value. Use Dutch error messages throughout.

# Q&A API Documentatie

## Server Actions (`app/qa/actions.ts`)

All actions require an authenticated user session. Unauthenticated calls return `{ error: 'Niet ingelogd' }`.

### createQuestion(formData: FormData)

**Returns:** `{ id: string } | { error: string }`

| Field | Type | Validatie |
|-------|------|-----------|
| title | string | min 10 tekens |
| body  | string | min 20 tekens |

### createAnswer(formData: FormData)

**Returns:** `{ id: string } | { error: string }`

| Field | Type | Validatie |
|-------|------|-----------|
| question_id | string (UUID) | geldig UUID |
| body        | string        | min 10 tekens |

### createReply(formData: FormData)

**Returns:** `{ id: string } | { error: string }`

| Field | Type | Validatie |
|-------|------|-----------|
| answer_id | string (UUID) | geldig UUID |
| body      | string        | min 5 tekens |

## Read Functions (`lib/qa/queries.ts`)

Async functions for use in Server Components. All functions throw on unexpected DB errors.

### getQuestions(limit?: number): Promise<Question[]>

Returns questions sorted by `created_at DESC`. Default limit: 20.

### getQuestionById(id: string): Promise<Question | null>

Returns a single question or `null` if not found.

### getAnswersByQuestionId(questionId: string): Promise<Answer[]>

Returns answers for a question sorted by `created_at ASC` (chronological).

### getRepliesByAnswerId(answerId: string): Promise<AnswerReply[]>

Returns replies for an answer sorted by `created_at ASC` (chronological).

## TypeScript Types (`lib/qa/types.ts`)

```typescript
type Question = { id, user_id, title, body, created_at, updated_at }
type Answer = { id, question_id, user_id, body, created_at, updated_at }
type AnswerReply = { id, answer_id, user_id, body, created_at, updated_at }
```

## Foutafhandeling

- Unauthenticated: `{ error: 'Niet ingelogd' }`
- Validation failure: `{ error: '<Dutch validation message>' }`
- DB/unexpected error: `{ error: 'Er is iets misgegaan. Probeer het opnieuw.' }`
- No stack traces or internal details are exposed to callers.
