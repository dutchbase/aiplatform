  <name>Task 1: Create GitHub Actions CI workflow</name>
  <files>.github/workflows/ci.yml</files>
  <action>
    Create the directory `.github/workflows/` and the file `.github/workflows/ci.yml`.

    The workflow must:
    - Name: "CI"
    - Trigger on: `push` to `main` branch AND `pull_request` targeting `main` branch
    - Use a single job named `lint-and-build` running on `ubuntu-latest`
    - Steps in order:
      1. `actions/checkout@v4` (no extra config needed)
      2. `actions/setup-node@v4` with `node-version: '20'` and `cache: 'npm'`
      3. `npm ci` (clean install — not `npm install`, to use package-lock.json exactly)
      4. `run: npm run lint` with name "Lint"
      5. `run: npm run build` with name "Build" — this validates TypeScript compilation and Next.js build output

    The `npm run build` step requires the Supabase env vars to be present or the build may fail. Add environment variables to the build step using dummy/placeholder values that allow the build to proceed without a real Supabase connection. Specifically, set these as `env:` under the build step:
    ```
    NEXT_PUBLIC_SUPABASE_URL: https://placeholder.supabase.co
    NEXT_PUBLIC_SUPABASE_ANON_KEY: placeholder-anon-key
    NEXT_PUBLIC_BASE_URL: https://example.com
    ```
    These placeholders allow the TypeScript build to succeed (env vars are present) without requiring actual Supabase credentials in GitHub Actions secrets.

    Do NOT add `SUPABASE_SERVICE_ROLE_KEY` to the CI env — it is only needed at runtime for server-side admin operations, not at build time.

    Final workflow structure example:
    ```yaml
    name: CI

    on:
      push:
        branches: [main]
      pull_request:
        branches: [main]

    jobs:
      lint-and-build:
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4
          - uses: actions/setup-node@v4
            with:
              node-version: '20'
              cache: 'npm'
          - run: npm ci
            name: Install dependencies
          - run: npm run lint
            name: Lint
          - run: npm run build
            name: Build
            env:
              NEXT_PUBLIC_SUPABASE_URL: https://placeholder.supabase.co
              NEXT_PUBLIC_SUPABASE_ANON_KEY: placeholder-anon-key
              NEXT_PUBLIC_BASE_URL: https://example.com
    ```
  </action>
  <verify>
    - File exists at `.github/workflows/ci.yml`
    - File contains `on:` with `push` and `pull_request` triggers
    - File contains `npm ci`, `npm run lint`, and `npm run build`
    - File contains placeholder env vars under the build step
    - File contains `actions/checkout@v4` and `actions/setup-node@v4`
    - YAML is valid (no tabs used for indentation — use spaces only)
  </verify>

       6. Click "Deploy" — Vercel runs `npm run build` automatically
