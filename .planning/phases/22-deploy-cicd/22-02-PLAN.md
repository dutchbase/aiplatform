<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Connect repo to Vercel and set production environment variables</name>
  <what-to-do>
    Complete these steps in the Vercel dashboard. Claude cannot perform these steps without a Vercel API token.

    **Step 1 — Create/log into Vercel account:**
    Go to https://vercel.com and log in or create a free account.

    **Step 2 — Import the GitHub repository:**
    1. Click "Add New..." -> "Project"
    2. Click "Import Git Repository"
    3. Find and select the `aiassistentplatform` repository
    4. Vercel auto-detects Next.js — leave Framework Preset as "Next.js"
    5. Do NOT click Deploy yet

    **Step 3 — Set environment variables (before first deploy):**
    In the "Environment Variables" section during import (or later in Settings -> Environment Variables):

    | Variable | Value | Environment |
    |----------|-------|-------------|
    | `NEXT_PUBLIC_SUPABASE_URL` | Your Supabase project URL | Production, Preview, Development |
    | `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Your Supabase anon/public key | Production, Preview, Development |
    | `SUPABASE_SERVICE_ROLE_KEY` | Your Supabase service_role key | Production only (KEEP SECRET) |
    | `NEXT_PUBLIC_BASE_URL` | Your Vercel domain e.g. `https://aiassistentplatform.vercel.app` | Production |

    Find Supabase values at: Supabase Dashboard -> Project Settings -> API

    For `NEXT_PUBLIC_BASE_URL`: use the Vercel-assigned domain for now (you can update this when you add a custom domain). The Vercel domain looks like `https://yourproject.vercel.app`.

    **Step 4 — Deploy:**
    Click "Deploy". Vercel will run `npm run build`. The first deploy takes 2-3 minutes.

    **Step 5 — Note the production URL:**
    After deploy completes, note the production URL shown by Vercel (e.g. `https://aiassistentplatform.vercel.app`).
  </what-to-do>
  <resume-signal>
    Type "deployed" followed by your production URL (e.g. "deployed https://aiassistentplatform.vercel.app") — or describe any errors that occurred.
  </resume-signal>
</task>

<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Verify production site is functional</name>
  <what-built>
    The Next.js application is now deployed on Vercel with production environment variables. This checkpoint verifies the live site works end-to-end.
  </what-built>
  <how-to-verify>
    Open your production URL and check the following:

    1. **Homepage** (`/`) — loads correctly, shows Dutch content, no errors in browser console
    2. **OpenClaw section** (`/openclaw`) — navigation cards visible, no blank page
    3. **Tutorials** (`/openclaw/tutorials`) — tutorial list loads
    4. **Blog** (`/blog`) — blog posts visible
    5. **Q&A** (`/qa`) — questions list loads (may be empty, that is fine)
    6. **Login page** (`/login`) — form is visible and functional
    7. **Auth test** — log in with a valid Supabase user (if you have one in production) or register a new account and verify email confirmation works
    8. **Sitemap** (`/sitemap.xml`) — returns XML, no error page
    9. **RSS feed** (`/feed.xml`) — returns RSS XML with correct production domain (not localhost)

    Also verify automatic deployments work:
    - Make a small change to any file (e.g. add a comment)
    - Commit and push to `main`
    - Check the Vercel dashboard — a new deployment should start automatically within ~30 seconds
  </how-to-verify>
  <resume-signal>
    Type "verified" if everything works — or describe which checks failed and any error messages seen.
  </resume-signal>
