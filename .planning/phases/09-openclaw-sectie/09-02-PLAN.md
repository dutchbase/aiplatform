  <action>
Create `components/openclaw/openclaw-nav.tsx` as Server Component with:

**Purpose:**
Reusable navigation component for all OpenClaw section pages (overview, installatie, tutorials, use-cases, nieuws).

**Component signature:**
```typescript
interface OpenClawNavProps {
  currentPath: string // e.g., "/openclaw", "/openclaw/installatie"
}

export function OpenClawNav({ currentPath }: OpenClawNavProps)
```

**Navigation structure:**
Horizontal tab-style navigation with 5 links:
1. **Overzicht** → `/openclaw`
2. **Installatie** → `/openclaw/installatie`
3. **Tutorials** → `/openclaw/tutorials`
4. **Use Cases** → `/openclaw/use-cases`
5. **Nieuws** → `/openclaw/nieuws`

**Active state logic:**
- Compare `currentPath` with each link's href
- Active link: different styling (e.g., border-b-2, font-bold, text-foreground)
- Inactive links: muted styling (text-muted-foreground, hover:text-foreground)

**Styling:**
- Horizontal layout: flex with space-x-6
- Border bottom on container: border-b border-border
- Active link: border-b-2 border-primary, text-foreground, font-semibold
- Inactive link: text-muted-foreground, hover:text-foreground transition
- Padding: py-2 for links, mb-8 for container (spacing below nav)
- Use Link from next/link
- Semantic tokens ONLY

**Implementation notes:**
- Server Component (no 'use client')
- Links array for easy maintenance:
  ```typescript
  const links = [
    { href: '/openclaw', label: 'Overzicht' },
    { href: '/openclaw/installatie', label: 'Installatie' },
    // ...
  ] as const
  ```
- Use Link component, not Button (this is horizontal tabs, not button-style nav)
- Export as named export: `export function OpenClawNav`
