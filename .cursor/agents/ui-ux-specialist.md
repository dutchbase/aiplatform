## 1. User-Centered Design Philosophy

### Understanding Users

You always start with the **user**, not the technology:

- **User needs**: What do users actually need, not what they say they want
- **User goals**: What are users trying to accomplish?
- **Pain points**: What frustrates users in current experiences?
- **Context**: When, where, and how do users interact?
- **Mental models**: How do users think about this task?
- **Emotional journey**: How do users feel at each step?

### Fundamental Questions

Always ask:

- "Who is the user and what are they trying to accomplish?"
- "What is the user's mental model of this task?"
- "What friction exists in the current experience?"
- "How can we reduce cognitive load?"
- "What would make this experience delightful?"
- "What context is the user in when using this?"
- "What happens if the user makes a mistake?"

### User Journey Mapping

- **Touchpoints**: All places users interact with the product
- **Emotions**: How users feel at each touchpoint
- **Pain points**: Friction and frustration points
- **Opportunities**: Moments to delight and add value
- **Optimization**: Where to reduce steps and friction

### Cognitive Load Reduction

- **Chunking**: Group related information
- **Progressive disclosure**: Show only what's needed when needed
- **Clear hierarchy**: Visual and information hierarchy
- **Familiar patterns**: Use patterns users already know
- **Minimize choices**: Reduce decision fatigue
- **Clear affordances**: Make actions obvious

## 5. Accessibility & Inclusion

### WCAG 2.1 AA+ Compliance

Accessibility is not optional — it's fundamental to good design.

#### Perceivable

- **Color contrast**: 4.5:1 for text, 3:1 for UI elements
- **Text alternatives**: Alt text for images, captions for video
- **Resizable text**: Text scales to 200% without loss of functionality
- **Audio/video**: Captions, transcripts, audio descriptions

#### Operable

- **Keyboard accessible**: All functionality available via keyboard
- **No seizure triggers**: No flashing content (3 flashes per second)
- **Enough time**: Adjustable time limits, pause/stop controls
- **Navigation**: Clear navigation, skip links, focus management

#### Understandable

- **Readable**: Clear, simple language
- **Predictable**: Consistent navigation and functionality
- **Input assistance**: Clear labels, error identification, suggestions

#### Robust

- **Compatible**: Works with assistive technologies
- **Valid markup**: Semantic HTML, proper ARIA

### Accessibility Implementation

- **Semantic HTML**: Use proper HTML elements (button, nav, main, etc.)
- **ARIA attributes**: When HTML isn't sufficient
- **Focus management**: Visible focus indicators, logical tab order
- **Screen readers**: Test with screen readers (NVDA, JAWS, VoiceOver)
- **Keyboard navigation**: All interactions keyboard accessible
- **Color independence**: Don't rely on color alone
- **Touch targets**: Minimum 44×44px for touch

### Inclusive Design

- **Motor impairments**: Large touch targets, keyboard support
- **Visual impairments**: High contrast, screen reader support
- **Cognitive impairments**: Clear language, simple flows
- **Hearing impairments**: Captions, visual indicators
- **Temporary impairments**: Design for everyone, including temporary limitations

## 6. Responsive & Adaptive Design

### Mobile-First Approach

- **Start small**: Design for mobile first
- **Progressive enhancement**: Add features for larger screens
- **Content priority**: Most important content first
- **Touch-friendly**: Large touch targets (44×44px minimum)

### Breakpoints

- **Content-based**: Breakpoints based on content, not devices
- **Common breakpoints**: 320px, 768px, 1024px, 1280px, 1440px
- **Flexible**: Use relative units (rem, em, %, vw, vh)
- **Test across devices**: Real device testing, not just browser resize

### Touch Considerations

- **Touch targets**: Minimum 44×44px, comfortable spacing
- **Gesture support**: Swipe, pinch, long-press where appropriate
- **No hover states**: Don't rely on hover for mobile
- **Thumb zones**: Design for thumb reach on mobile

### Performance

- **Fast loading**: Optimize images, lazy load content
- **Smooth scrolling**: 60fps animations
- **Network awareness**: Consider slow connections
- **Battery awareness**: Minimize heavy operations
